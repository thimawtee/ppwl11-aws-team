## Overview
Proyek ini merupakan implementasi arsitektur berbasis monorepo, yaitu pendekatan pengelolaan kode di mana beberapa layanan atau aplikasi disimpan dalam satu repository terpusat. Pendekatan ini mempermudah manajemen dependensi, kolaborasi tim, dan integrasi antar layanan. 

## Fase 1 ✅

Pada fase ini:

- Berhasil menginisialisasi dan mengkonfigurasi layanan AWS
- Menghubungkan repository dengan environment AWS
- Menyiapkan infrastruktur awal untuk deployment sistem

Hasil:
📸 Dokumentasi :[ *Paramter_Store*]([url](https://drive.google.com/file/d/1rb5EeO3kEz31DKTXcdB0j2BvOry_02p6/view?usp=sharing)
)

## Fase 2 ✅

Pada fase ini:

- Berhasil melakukan monitoring penggunaan resource AWS
- Menampilkan estimasi biaya layanan yang digunakan
- Mengontrol penggunaan resource agar tetap efisien

Hasil:
📸 Dokumentasi :[ *Budget & Cost*]([url](https://drive.google.com/file/d/1PFBtKQTLJ9fGNf6WIdh6insSN-UZrp_c/view?usp=sharing)
)

## Fase 3 ✅

Pada fase ini:

- Berhasil membuat database instance menggunakan RDS
- Database berada dalam status available/active
- Database siap digunakan untuk kebutuhan backend

Hasil:
📸 Dokumentasi :[ *RDS_Console*]([url](https://drive.google.com/file/d/12A4988sk0QdwaWEkxhHPv6dhPZlNLyrm/view?usp=sharing)

## Fase 4 ✅

Pada fase ini:

- Berhasil membuat fungsi backend menggunakan Lambda
- Fungsi dapat dijalankan tanpa server (serverless)
- Backend mampu memproses request dengan baik

Hasil:
📸 Dokumentasi :[ *Lambda-Backend*]([url](https://drive.google.com/file/d/1R4SYVjOR4EwiHegZmYBGr0k9qqz_nqf9/view?usp=sharing)

## Fase 5 ✅
